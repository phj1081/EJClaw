# RUNEFALL Prototype Asset Request

현재 프로토타입은 모든 아트를 색 블록/라벨로 대체합니다. 다음 단계에서 실제 모바일 게임처럼 보이게 하려면 아래 에셋이 필요합니다.

## 우선순위 A: 코어 플레이 검증용

- 플레이어 캐릭터 픽셀 스프라이트 4종: 대기/이동/피격, 4방향 또는 8방향, 32x32 또는 48x48 권장
- 적 픽셀 스프라이트 3종: 소형 근접, 원거리, 탱커
- 투사체 6종: 화염, 중력, 소환, 관통, 연쇄, 수호 태그별 기본 탄
- 이펙트 6종: 히트, 사망, 대시, 전환, 레벨업, 융합 발동
- 전투 배경 타일 1종: 밝은 하이 판타지 폐허/초원 계열, 16:9 가로 반복 가능
- UI 아이콘: 골드, 젬, 소재, 체력, 경험치, 대시, 스킬, 일시정지

## 우선순위 B: UX/메타 화면용

- 캐릭터 초상화 6종: 파티 패널/편성/정산 공용
- 무기 아이콘 10종과 태그 배지 6종
- 장비 아이콘: 무기/갑옷/장신구/공용 슬롯/전용 슬롯
- 메인 거점 배경 1종: 파티 4인이 서 있는 밝은 픽셀 아트 허브
- 버튼/패널 9-slice 스타일: 어두운 반투명 패널, 강조 CTA, 비활성 버튼

## 우선순위 C: 모바일 완성도용

- 코스튬 샘플 2~4종
- 보스 스프라이트 1종과 등장 이펙트
- 사운드: 기본 공격, 피격, 대시, 스킬, 레벨업, 융합, 승리/패배
- BGM: 메인 화면 1곡, 전투 1곡

## 권장 규격

- 픽셀 그리드: 16px 또는 32px 기준으로 통일
- 캐릭터 팔레트: 캐릭터당 4~8색 중심
- 배경 대비: 투사체와 적 실루엣이 묻히지 않도록 채도 낮춤
- 파일 형식: PNG 스프라이트시트, 투명 배경
- 네이밍 예: `hero_luna_run_32.png`, `fx_fusion_fire_gravity.png`
